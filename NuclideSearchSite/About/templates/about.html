About NuclideSearch

Being a long time user of the Lund/LBNL Nuclear Data Search site, I found that the design and usability did not meet up to the standards of modern websites. When I found the database behind the site freely available (here), I decided that it would be possible to replicate and improve the functionality of the Lund/LBNL-site with relative ease. Most if not all the functionality of the Lund/LNBL-site was re-implemented though the layout was changed significantly. In addition, the search system of this site is a vast improvement over the Lund/LBNL-site. Some smaller improvement are also provided such as nuclide history and better navigation between mother and daughter nuclides. If you have additional suggestions on improving the site, please

Technical details

The site uses the Django-framework for all the backend functionality. The nuclear data is stored in a SQLite database. For styling, the Bootstrap front-end framework is used. To provide tables that can be sorted without reloading the page bootstrap-sortable is used. Auto completion is provided by the typeahed.js library. The site does not keep track of your searches. This data is instead stored by your browser using the local storage functionality introduced in HTML 5.
